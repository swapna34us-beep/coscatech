What Is This System?
The GRT-HCI framework and what it tests
GRT (Goals, Rules, Thresholds) is a structural governance framework that treats AI decision pipelines as controllable dynamical systems. Instead of asking "is this output acceptable?" it asks: under what frame, subject to which rules, at what thresholds was this output generated, and where does human judgment enter the loop?
The Experiment
Two groups built the same AI system (supplier recommendation for indirect procurement) from the same 10,000-transaction dataset under different governance processes:

Control Group: Standard ML. No governance document. Target: 15% cost reduction.
Treatment Group: Write a complete Governance Reference Document (GRD) — Goals, Rules, Thresholds — before touching data. Build only what the GRD authorizes.

The experiment tests 5 claims across 19 metrics organized in 3 layers (Input, Process, Output). Two complementary control-theoretic frameworks — Laplace-domain (intra-regime stability) and Lyapunov switched-system (inter-regime transitions) — diagnose threshold miscalibrations.

Hidden Fields (what the model cannot see)
The full dataset has 5 fields the model never sees — these represent the buyer's situated knowledge:
diversity_certified (protected attribute), relationship_years, buyer_trust_score, disruption_resilience, strategic_alignment

The GRT framework treats these as residue — the gap between what the model knows and what the human knows. The HCI score measures the value of that gap.

Stage 3: Threshold Calibration
Testing whether governance thresholds fire at the right rate — with Laplace and Lyapunov diagnostics
Each threshold acts as a controller gain (paper §4.4). The engine fires each threshold against all 10,000 transactions, computes the fire rate, and compares against the GRD-declared target band.

Laplace interpretation: Under-firing = over-damped (sluggish). Over-firing = marginally stable (oscillation risk).
Lyapunov interpretation: Checks whether the switched system can cross regime boundaries stably.
4
Stage 4: Human Contribution Index
Measuring where, when, and how much human judgment enters the decision loop
HCI = (H_F × H_D × H_R)^(1/3) — geometric mean, non-compensable. If any sub-index is zero, HCI is zero. A system with strong frame authority but no residue surfacing is not "partially governed."

Stage 6: Claim Verdicts
What the experiment proved
Each claim is evaluated against its primary metrics. CONFIRMED = all primary metrics favor Treatment. PARTIALLY CONFIRMED = directionally favorable but not strongest outcome. PENDING = requires post-deployment data.
What This Means
The GRT framework produces measurable governance signal at an accuracy cost of -0.2%. Writing a GRD before building changes what gets built (C2), surfaces problems earlier (C1), changes the deployment decision (C3), and the cost is worth it (C5). The framework metrics themselves produce real signal (C4 pending only because M10-M13 need live data).

Three thresholds need recalibration before a live buyer experiment. The knowledge roster is defined. The engine is ready.
